Multi-AZ Kubernetes Cluster with NFS Storage

This diagram shows the complete AWS infrastructure for a highly available Kubernetes cluster with NFS-based shared storage across multiple Availability Zones.

Architecture Notes

The NFS storage server provides shared file system access for all nodes in the Kubernetes cluster across multiple Availability Zones. This setup includes redundant master nodes, distributed etcd cluster, and a load balancer for high availability.

Note: Some connections (like SSH to all instances and cross-AZ etcd communication) are simplified in the diagram for clarity and readability.

How NFS Storage Enables Cross-AZ Access

The NFS storage server provides a centralized file system that can be mounted by all nodes in the Kubernetes cluster across different Availability Zones. This setup ensures consistent storage access for persistent volumes and shared data.

Key benefits:

  • Centralized storage server in AZ 1c
  • Network file system access across all AZs
  • High availability through redundant infrastructure
  • Suitable for Kubernetes persistent volumes
  • Cost-effective compared to managed services
